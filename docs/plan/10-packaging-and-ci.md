# 10 - Packaging & CI Releases

## Goal

Package ViberWhisper as proper desktop applications (macOS `.app`/`.dmg`, Windows `.msi`) and automate cross-platform builds via GitHub Actions CI. Addresses issue #42.

## Requirements

1. **macOS packaging**: Binary wrapped in `.app` bundle → packaged as `.dmg`
2. **Windows packaging**: Binary packaged as `.msi` installer
3. **CI automation**: GitHub Actions builds and publishes releases on tag push
4. **Universal macOS binary**: Support both x86_64 (Intel) and aarch64 (Apple Silicon)
5. **Raw binaries available**: GitHub Releases include both installers and raw binaries

## Tech Choice

### macOS: `cargo-bundle` + `hdiutil`

- `cargo-bundle` generates `.app` bundle from Cargo.toml metadata
- `hdiutil` (built-in macOS tool) creates `.dmg` from `.app` — no Node.js dependency needed
- Works well with platform-specific deps (`cocoa`, `objc`)

### Windows: `cargo-wix`

- Generates `.msi` installer via WiX Toolset
- Template-based, pulls metadata from `Cargo.toml`
- Supports code signing (deferred)

### CI: GitHub Actions with native runners

- `macos-latest` runner for both x86_64 and aarch64 targets (cross-compile on same runner)
- `windows-latest` runner for x86_64-pc-windows-msvc
- `lipo` to create universal macOS binary
- Triggered on version tag push (`v*.*.*`)

### Why not other options?

- **Tauri bundler**: Overkill — project is pure Rust CLI, not a web app
- **NSIS for Windows**: cargo-wix is more Rust-native, less config overhead
- **cross-rs**: Not needed — native runners handle our targets fine, and platform-specific deps (cocoa) require actual macOS SDK

## Implementation

### 1. Cargo.toml Bundle Metadata

```toml
[package.metadata.bundle]
name = "ViberWhisper"
identifier = "com.b1indsight.viberwhisper"
icon = ["assets/icon-32x32.png", "assets/icon-128x128.png", "assets/icon-256x256.png"]
copyright = "Copyright (c) 2026 b1indsight"
osx_minimum_system_version = "11.0"
short_description = "Voice-to-text input tool"
```

### 2. macOS .app Bundle

`cargo-bundle` creates:
```
ViberWhisper.app/
├── Contents/
    ├── Info.plist          # Generated from Cargo.toml metadata
    ├── MacOS/
    │   └── viberwhisper    # Release binary
    └── Resources/
        └── AppIcon.icns    # App icon
```

Key Info.plist entries (auto-generated by cargo-bundle):
- `CFBundleIdentifier`: `com.b1indsight.viberwhisper`
- `CFBundleExecutable`: `viberwhisper`
- `LSUIElement`: `true` (menu bar app, no Dock icon)
- `NSMicrophoneUsageDescription`: "ViberWhisper needs microphone access for voice recording"

### 3. Windows MSI

```bash
cargo wix init    # Generates wix/main.wxs template (one-time)
cargo wix         # Builds target/wix/viberwhisper-0.1.0-x86_64.msi
```

WiX template in `wix/main.wxs` — customized from default to include:
- Start menu shortcut
- Desktop shortcut (optional)
- Uninstaller entry in Add/Remove Programs

### 4. GitHub Actions Workflow

File: `.github/workflows/release.yml`

**Trigger**: Push tag matching `v*.*.*`

**Jobs**:

#### Job 1: `build-macos`
Runner: `macos-latest`

Steps:
1. Checkout code
2. Install Rust stable + targets (`x86_64-apple-darwin`, `aarch64-apple-darwin`)
3. Install `cargo-bundle`
4. Build release for both targets
5. Create universal binary with `lipo`
6. Run `cargo bundle --release` for `.app`
7. Replace binary in `.app` with universal binary
8. Create `.dmg` with `hdiutil create`
9. Upload artifacts: `viberwhisper-macos-universal.dmg`, `viberwhisper-macos-x86_64`, `viberwhisper-macos-aarch64`

#### Job 2: `build-windows`
Runner: `windows-latest`

Steps:
1. Checkout code
2. Install Rust stable
3. Install WiX Toolset + `cargo-wix`
4. Build release for `x86_64-pc-windows-msvc`
5. Run `cargo wix` to create `.msi`
6. Upload artifacts: `viberwhisper-windows-x86_64.msi`, `viberwhisper-windows-x86_64.exe`

#### Job 3: `release`
Depends on: `build-macos`, `build-windows`

Steps:
1. Download all artifacts
2. Create GitHub Release with tag name
3. Upload all artifacts to release

### 6. Version Tagging Convention

- Semantic versioning: `v0.2.0`, `v0.3.0`, etc.
- Current version in `Cargo.toml`: `0.1.0`
- First release tag: `v0.2.0` (after packaging is implemented)
- Release flow: bump `Cargo.toml` version → commit → push tag → CI builds & publishes

## Implementation Steps

1. Create `assets/` directory with placeholder app icons
2. Add `[package.metadata.bundle]` section to `Cargo.toml`
3. Run `cargo wix init` to generate `wix/main.wxs` template
4. Create `.github/workflows/release.yml` with the 3-job pipeline
5. Test macOS build locally: `cargo bundle --release`
6. Test DMG creation locally: `hdiutil create -volname ViberWhisper -srcfolder target/release/bundle/osx/ViberWhisper.app -ov -format UDZO ViberWhisper.dmg`
7. Push tag to trigger CI and verify artifacts
8. Update `CLAUDE.md` with packaging commands
9. Update `changelog`

## Deferred

- **macOS code signing & notarization**: Requires Apple Developer account ($99/yr). Can use `apple-codesign` (Rust crate) when ready.
- **Windows code signing**: Requires code signing certificate. Can integrate with `cargo-wix --sign` when ready.
- **Auto-update mechanism**: Consider for future (e.g., Sparkle for macOS, WinSparkle for Windows).
- **Linux packaging**: .deb/.AppImage — add when there's demand.
